Describe what happens when you visit youtube.com?

When you enter "youtube.com" in your browser, the browser first performs a DNS lookup, similar to finding a phone contact in your address book youtube.com ---> +1 561-555-7689(it is fake, DO NOT CALL) to retrieve the IP address associated with the domain. Once the IP address is found, the browser sends an HTTP or HTTPS request to the server. The server then responds by sending the webpage, which the browser renders for you to view.

What are the components of a web page document?

  • DOCTYPE Declaration: The document starts with a DOCTYPE declaration (e.g., <!DOCTYPE html>) that tells the browser what version of HTML is being used. This helps the browser render the page properly.
  • HTML Element: The <html> element is the root of the document, wrapping everything inside. It tells the browser that this is an HTML document and usually includes the lang attribute to specify the language of the page.
  • Head Section: The <head> element contains meta information about the document, such as the title (<title>), character set (<meta charset="UTF-8">), and links to external resources like stylesheets and scripts. Itâ€™s not directly visible on the page, but crucial for proper functionality.
  • Body Section: The <body> element holds all the visible content of the webpage, such as text, images, links, forms, etc. This is the main part of the document that users interact with.
  • Scripts: JavaScript files, either inline or external, are loaded into the document using the <script> tag. They provide interactivity and dynamic functionality to the page.
  • Stylesheets: CSS can be linked externally or included inline within the <style> tag. These stylesheets control the appearance and layout of the page elements, making the page visually appealing and user-friendly.

What is the World Wide Web?

The World Wide Web (WWW) is a vast information space that allows users to access and navigate interconnected documents and resources over the Internet. It utilizes the HTTP (Hypertext Transfer Protocol) and consists of a collection of web pages, images, videos, and other multimedia content that are linked together through hyperlinks. The web enables users to retrieve, share, and interact with information using web browsers and provides a platform for countless applications and services.

Differences bewteen URI and URL?

  • URI (Uniform Resource Identifier): A URI is a generic string of characters that identifies a resource either by location, name, or both. It provides a way to refer to a resource, but it doesn't necessarily indicate how to access it. For example, a URI can represent a resource using its name, such as urn:isbn:0451450523 (which identifies a book using its ISBN). When your reference some paper, you got to use URI, like https://doi.org/10.1145/2783446.2783605, it is unique, that is why urI (identifier)
  • URL (Uniform Resource Locator): A URL is a specific type of URI that provides the means to access a resource over the Internet. It includes the protocol (such as http or https), the domain name (like wkusci.com), it is used to specify how to access a particular resource.
  • Differences:
    • Scope: All URLs are URIs, but not all URIs are URLs. URLs specify a way to retrieve a resource, while URIs can simply identify it.
    • Structure: URLs provide a complete address that includes the protocol and resource location, while URIs may not provide such information.

How does DNS work?

DNS is like the phone book of the internet, translating domain names into IP addresses, allowing browsers to access servers.

Analyze https://d2w.wkusci.com:80/blogs/have-a-nice-day/?invitation=4590098&recommend=232423#have-fun

https is the protocol or schema for transmitting data.

d2w.wkusci.com:80 is the authority, where d2w.wkusci.com is the domain and 80 is the port. Port 80 is the default for HTTP, and 443 for HTTPS. If your backend server provides services on port 6666, https://d2w.wkusci.com:6666/user/?id=12321 would be a complete URL where you access services provided on port 6666 of d2w.wkusci.com with parameters id=12321.

The scheme and authority are separated by ://. The colon separates the scheme from the next part of the URL, and // indicates the next part is the authority. One example of a URL without authority is a mail client mailto:foobar, which includes a scheme but no authority.

After the domain name is the path /blogs/have-a-nice-day/, ending with a slash (required when the URL represents a directory).

Following the path, the URL includes ?invitation=4590098&recommend=232423, which provides query parameters. The #have-fun at the end is a fragment identifier for navigating within the page.

What is a static server and what is a dynamic server?

A static server refers to a server that only provides static resources such as HTML, CSS, JavaScript, images, videos, etc. A dynamic server, on the other hand, can return different content based on the request. We call it "dynamic" because the application server updates the hosted files, such as html, before sending them to your browser via the HTTP server.
